# Media Downloader Telegram Bot — Design

Дата: 2026-07-09
Репозиторий кода: новый `Amar73/telegram-bots` (не `hermes-scripts` — тот зарезервирован за MARKO Lab pipeline)
Локальный чекаут: `/home/amar/Amar73/telegram-bots`
Бот: `@amardownloader_bot` (новый, через `@BotFather`, отдельно от тестового `AITGWatchdog_bot` и от `@AmarAiAgent_bot`)

## Контекст

Первая "реальная" (не демо/тестовая) задача, написанная напрямую в этой сессии, а не через MARKO Lab pipeline — пользователь явно выбрал этот путь (быстрее, у сессии уже есть весь контекст по серверу). Полное функциональное ТЗ было полностью сформулировано пользователем; этот документ фиксирует только архитектурные решения, которые ТЗ не покрывало (лимиты Telegram, хостинг, репозиторий).

## Функционал (из ТЗ пользователя)

Бот принимает прямые ссылки или пересланные сообщения со ссылками на YouTube (video/shorts) и Instagram (Reels/посты/stories), скачивает видео через `yt-dlp` и отправляет обратно в чат файлом. Команды `/start`, `/help`. Статусные сообщения («Скачиваю…», «Загружаю…», «Готово»). Понятные ошибки для неподдерживаемых/недоступных ссылок.

## Архитектурные решения (не покрыты исходным ТЗ)

- **Лимит размера файла: 50 МБ (облачный Bot API), не 2 ГБ.** Исходное ТЗ ссылалось на лимит 2 ГБ — это верно только для self-hosted Local Bot API Server (требует `api_id`/`api_hash` с my.telegram.org и отдельный процесс). Решено остаться на обычном облачном Bot API и подбирать формат/качество через `yt-dlp` под 50 МБ; если под лимит ничего не подобрать — сообщить пользователю честно, не пытаться обойти лимит.
- **Репозиторий и структура:** новый GitHub-репозиторий `Amar73/telegram-bots` (не `hermes-scripts`) — код, написанный напрямую в этой сессии, живёт отдельно от кода, который пишет MARKO Lab pipeline. Этот бот — в поддиректории `media-downloader/` внутри репозитория (репозиторий рассчитан на несколько ботов со временем).
- **Хостинг:** тот же сервер `hermes` (193.228.139.46), но **под пользователем `amar`**, не `paperclip` — бот не имеет отношения к Hermes/Paperclip-агентам, отдельный systemd-сервис со своим venv, не завязан на инфраструктуру MARKO Lab.
- **Токен:** новый, выделенный (`@amardownloader_bot`), не переиспользует тестовый/агентский токены.

## Зависимости на сервере (нужно установить)

- Python 3.10+ (уже есть системный `python3`, версия проверяется в плане)
- `yt-dlp` (pip)
- `ffmpeg` (системный пакет — нужен `yt-dlp` для мёрджа/перекодирования форматов и генерации миниатюр)

## Явно вне скоупа

- Локальный Bot API сервер / поддержка файлов >50 МБ без даунскейла.
- Docker-обёртка (упомянута в ТЗ как «по желанию» — не делаем сейчас, достаточно systemd).
- Whitelist пользователей/авторизация — бот личный, single-user, не рассматриваем как публичный сервис.
